# Session 2026-04-03 — trib-memory Refactoring & Tuning

## What Was Done

### Pipeline Refactoring
- Dropped all legacy tables (facts/tasks/signals/profiles/entities/relations/propositions)
- 3-field classification schema: topic, element, importance
- cycle1: pending candidates → CSV classification via gpt-5.4-mini
- LLM calls: worker child IPC for all 3 plugins (trib-memory/channels/search), `-p` removed
- sqlite-vec installed, vecEnabled=true

### Scoring Pipeline
- RRF (Reciprocal Rank Fusion) replacing additive BM25+cosine
- importance tags: rule/directive/decision/preference/incident/transient
- Power-law decay: `1/(1+age/30)^0.3`, tag_factor on loss (rule=0 never decays)
- importance_boost: 0.05 (intentionally small)
- exact match bonus: base*0.2 when query substring found in content
- classification cap: max 2 per search result
- semantic_bonus removed (was causing classification to overpower episodes)
- time_factor/state_factor/language_factor removed from search (decay is separate concern)

### Context.md
- Core Memory: rule/directive/preference/decision (top 5, permanent)
- Bot: user-managed bot.md (unchanged)
- Recent Decisions: incident etc. (top 5, rotating)
- Recent: last 5 turns user+assistant ping-pong

### Benchmarks
- natural-80: hit@1=81.3% hit@3=96.3% matched=78/80
- natural-40: hit@1=85% hit@3=100% matched=40/40
- Benchmark uses keyword-based expected (schema-independent)

### Temporal Parsing
- Korean: 어제, 3월 30일, 방금, 지난 수요일, N주전, 엊그제
- Dateparser (Python): available as opt-in, JS parsing is primary

### Dead Code Cleanup
- Removed: memory-decay-utils.mjs, memory-candidate-utils.mjs
- Cleaned: memory-maintenance-store, memory-recall-store, memory-ranking-utils
- All legacy table queries removed
